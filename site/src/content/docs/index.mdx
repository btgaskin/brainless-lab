## An experimentation platform

BrainlessLab is a runnable lab for **minimal neural collectives**. Each collective contains
simple, homeostatic, neuron-like nodes. It has no trained readout and no controller. It is for
researchers who study real neurons or cells and want a small, designed collective they can run.

The anchor question is why the Falandays node works. You can test that question, design
variations, and compare them. Each comparison tests for a capacity, such as responsiveness,
coordination, or memory. A score indicates that the capacity is present. It does not rank
designs in any absolute sense. These tests span the cellular cognition of a single node and
the collective intelligence of a swarm.

The collective self-organises toward its own viability, not toward our tasks. BrainlessLab
measures whether the collective can be made to perform a task.

You can add tasks and environments. The library provides abstractions for bodies, receptors
and effectors, sensor placement, parameters, and reservoir structure. Modelling stays under
biologically plausible constraints. The compute core is low-code and headless. It runs on a
laptop and does not require a GPU.

:::note[Ships with two agent skills]
The repository includes two agent skills. One covers Julia. The other covers BrainlessLab
itself.
:::

## What you run

These four tools cover comparisons, detailed profiles, parameter sweeps, and composed studies.

  <Card title="bench" icon="list-format">
    Run a set of node models across a grid of tasks. Compare the scores and rankings. Each
    score indicates the capacity tested by its task.
    [Benchmark →](/tooling/#the-commands)
  </Card>
  <Card title="profile" icon="magnifier">
    Examine one node in detail. Measure its internal signatures, the region where it works,
    and how it fails.
    [Profile →](/tooling/#the-commands)
  </Card>
  <Card title="sweep" icon="random">
    Scan parameters and configurations. Map the viable region and identify how the node fails

  </Card>
