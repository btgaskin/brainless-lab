## A platform for comparative neural design

BrainlessLab connects four parts of an experiment:

```text
node model + embodiment + task + evaluation protocol
```

The platform begins with a fixture-validated implementation of the Falandays
homeostatic reservoir and two core tasks from the source study: object Tracking and Pong.
These provide a small common test set for asking how a node design behaves when its
sensorimotor setting changes.

Performance is evidence about a declared composition. It is not a universal measure of
intelligence. A low score can still identify a missing capacity, an unsuitable body, or a
parameter regime that does not support the task. That makes the benchmark useful for both
evolving existing parameters and designing new local mechanisms, such as dendritic
compartments.

## One runtime, five research operations

Use `simulate` to inspect one closed-loop run. Use a typed plan when the work must be
repeatable:

- `ProfilePlan` characterises one composition;
- `SweepPlan` maps declared parameters;
- `AblationPlan` disables registered mechanisms;
- `EvolutionPlan` develops a registered parameter set;
- `BenchmarkPlan` compares conditions within tasks.

An `ExperimentSpec` groups named conditions and operation plans into a versioned scientific
protocol. Every operation writes the same record structure: submitted and resolved
settings, named random streams, raw tables, summaries, provenance, checksums, and an HTML
report.

The compute package is headless, runs on a laptop, and does not require a GPU. Optional
Makie extensions provide figures and interactive views.
