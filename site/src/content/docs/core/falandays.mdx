Use `node=:falandays` for the canonical BrainlessLab reference:

```julia
sim = simulate(:tracking; node=:falandays, ticks=1000, seed=11)
```

This implementation follows the model in Falandays, Yoshimi, Warren, and Spivey,
[“A potential mechanism for Gibsonian resonance”](https://doi.org/10.1007/s11571-023-09988-2).

## Node update

Each node is a leaky integrate-and-fire unit. Its activation receives sensory current and
recurrent input from spikes at the previous neural step:

$$
a_i(t) \leftarrow a_i(t-1)(1-\lambda)
  + \sum_r W^{\mathrm{in}}_{ri}R_r(t)
  + \sum_j W_{ji}(t)s_j(t-1).
$$

The node spikes when its activation crosses a multiple of its local target:

$$
s_i(t) = \mathbb{1}[a_i(t) \ge \mu T_i(t)].
$$

After a spike, the implementation subtracts the threshold from the activation. It does not
set the activation to zero. The local error is then

$$
e_i(t) = a_i(t) - T_i(t).
$$

If \(k_i(t)\) presynaptic neighbours spiked at the previous step, the unsigned reference
update changes each active incoming recurrent weight by

$$
W_{ji}(t+1) = W_{ji}(t) -
  \eta_W\frac{e_i(t)}{k_i(t)}.
$$

The target activity level changes as

$$
T_i(t+1) = \max\left(T_i(t) + \eta_T e_i(t), T_{\min}\right).
$$

Both changes occur during behaviour. There is no separate task-loss training phase, but
the reservoir does adapt online.

## Canonical task settings

Tracking, Pong, and Wall use settings derived from the authors' task implementations:

```julia
falandays_paper_config(:tracking)
falandays_paper_config(:pong)
falandays_paper_config(:wall)
```

The settings differ in node count, receptor width, input amplitude, target learning rate,
and weight initialisation. A named node therefore does not imply one numerical
parameterisation across tasks. The resolved composition and evaluation record are the
method.

BrainlessLab uses Tracking and Pong as the initial core task set. Wall remains available
for experimental work but is not part of that core comparison.

## Validation boundary

The canonical implementation is compared with authors-derived, 48-step reference
trajectories for Tracking, Pong, and Wall. The checks cover activation, targets, spikes,
errors, and final recurrent weights at the declared tolerances.

This is trajectory-level fixture validation. It does not establish:

- closed-loop behavioural replication of every result in the article;
- fidelity for every body, task, or experimental variant;
- biological validity of the node model;
- general performance beyond the declared composition.

Keep three questions separate:

1. Does the implementation match the declared reference trajectories?
2. How does a composed agent perform under a declared task and evaluation protocol?
3. What scientific claim is supported by the controls and independent replications?

## Variants and new mechanisms

Experimental variants alter recurrent structure, signs, delays, dendritic eligibility,
sensory noise, or endogenous drive. Compare them while holding unrelated parts of the
composition and evaluation fixed.

Use a parameter or registered mechanism when the update interface stays the same. Add a
new `Reservoir` subtype when state, stepping, plasticity, or readout changes materially.

Read [Reservoirs and node models](/core/reservoirs/) for preparation rules and
[Design a study](/core/design-study/) before comparing variants.
