The **task** you pick fixes the `Environment` and the `(n_receptors, n_effectors)` the
whole stack is built to â€” it shapes the assembly *and* supplies the score. Single-agent
tasks use a `PassthroughBody` over a `TaskWorld`; the swarm uses a `VENBody` in a
`TorusEnvironment` / `ForageEnvironment`.

## Single-agent tasks

| Task | R | E | senses / acts with |
|---|---:|---:|---|
| `:wall` | 2 | 2 | two ray-cast wall sensors; differential wheel speeds |
| `:tracking` | 62 | 2 | two eyes Ă— 31 Gaussian bearing sensors; eye-rotation command |
| `:pong` / `:pong_hitrate` | 46 | 2 | ball-bearing bins; paddle vote/differential |
| `:cartpole` (+ hard/swingup/long) | 8 | 2 | 4 state dims Ă— 2 polarities; binary force vote |

Effector semantics are **intentionally non-uniform**: channel 1 in Pong is not the same
physical quantity as channel 1 in cartpole. `normalized_score(task, raw)` maps each
task's raw score onto $[0,1]$ via a per-task floor/ceiling â€” for optimizer bookkeeping,
not a universal unit. Don't compare raw scores across tasks.

## The swarm

`VENBody` turns bearing-vision into receptors and VEN kinematics out of effectors: **62**
bearing sensors (two eyes Ă— 31 angles) padded to **64** inputs; **3** effectors (`e3`
forward acceleration, `e2âˆ’e1` heading acceleration). Coupling is **vision** â€” agents
influence each other only by being seen, so the sensor geometry *is* the interaction
topology; collective order is emergent, not imposed.

## The foraging experiment

`:forage` adds a **source** the agents can find, sensed via a *second* bearing-vision
bank (its own gain) alongside the conspecific bank. The headline manipulation is
**social vs blind**:

- **social** â€” agents see each other *and* the source;
- **blind** â€” the same `N` agents see the source but are blind to each other (and don't collide).

The scientific question: does social vision speed convergence to the source â€” i.e. do
agents use each other's presence as a guide? Metrics: mean distance-to-source, fraction
within a capture radius, and time-to-first-arrival.

```julia
simulate(:forage; node=:falandays_base, n_agents=12, conspecific_vision=true)   # social
simulate(:forage; node=:falandays_base, n_agents=12, conspecific_vision=false)  # blind
```
