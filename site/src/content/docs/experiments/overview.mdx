This section contains studies and working notes built with BrainlessLab. It is not a list of
equally established “published results.” Every page declares an evidence record in its
front matter:

- `status`: exploratory, tuned, frozen, confirmed, promoted, or retired;
- `id` and protocol version;
- randomization unit and number of independent blocks;
- primary endpoint;
- artifact path;
- the limitation that most constrains interpretation.

The labels follow the [Research workflow](/research-workflow/). A traceable run is not
automatically confirmatory: code and seeds may be recoverable even when the same data were
used to tune parameters and interpret the result.

```bash
julia --project=. experiments/run.jl --list
julia -t auto --project=. experiments/run.jl <name>
```

There are currently no pages labelled `frozen`, `confirmed`, or `promoted`. That is a useful
state, not a failure: the existing material generates hypotheses and tests the platform,
while the new evidence contract prevents exploratory work from silently becoming
confirmation. `frozen` means the protocol is fixed and the sealed outcomes remain unopened;
`confirmed` means that protocol has been executed on the sealed blocks.

## What belongs here

An analysis over one recorded run belongs in the analysis registry. A protocol that
coordinates conditions across many runs belongs under `experiments/`. Keep protocols on the
public API so core refactors do not become study-specific.

Before a study can be promoted, its bundle must include the frozen protocol, resolved
configuration, full code/environment provenance, disjoint seed ledger, per-block data,
declared contrasts and inferential unit, analysis version, summary data, figure inputs,
representative-selection rule, and checksums. See `experiments/README.md` for the repository
checklist.

:::note[Stable model does not mean confirmed study]
The fixture-validated `:falandays_base` reservoir can be used inside an exploratory task,
body, analysis, or protocol. Baseline implementation status and study evidence status are
separate.
:::
